# Интернет Магазин

### Выгода

- Переход на микросервисную архитектуру позволит распределить бизнес-функции по разным независимым сервисам;
- Появится возможность использования разных технологий подходящих непосредственно для каждого микросервиса;
- Увеличится скорость разработки и частота выкладки релизов, т.к. каждый микросервис можно развивать отдельно от других;
- Упростится создание нового функционала системы за счёт добавления новых микросервисов;
- Упростится масштабирование системы и повышение отказоустойчивости из-за отсутствия единой точки отказа;
- Микросервисная архитектура может выгодно отражать структуру организации, что будет повышать привлекательность организации в качестве работодателя (ИМХО).

### Подводные камни и проблемы

- Важно избежать образование наносервисов;
- Необходимо быть готовым к решению проблем разработки:
  - версионирование артефактов;
  - настройка конвейеров автоматической сборки и тестирования;
  - создание и поддержание документации(!!!).
- Необходимо быть готовым к решению проблем эксплуатации:
  - Сбор логов и метрик систем;
  - Управление конфигурацией и инфраструктурой.
Как вывод - для решения проблем разработки и эксплуатации может потребоваться расширения штата сотрудников и увеличение бюджета на инфраструктуру.
Перед переходом важно разработать подход к разделению системы на микросервисы, определить правила владения ими и способ межсервисного взаимодействия.
